# Headhunter Jira Source

## Data Available

Jira-sourced events carry:
- Embedded YAML execution plan in the `reason` field (domain, risk, steps with agent/mode/summary)
- Structured `jira_context` in evidence: issue_key, issue_url, summary, status, priority, components, labels
- The plan was generated by Claude analysis and approved by a human (status moved to "To Do")

## Routing

Follow the embedded YAML plan steps. Each step specifies `agent` and `mode`.
Do NOT infer routing from issue type or labels -- the plan already encodes the correct workflow.

The plan may reference any agent (qe, developer, architect, sysadmin) depending on the activity
label that generated it. Trust the plan.

## Triage Enrichment

During triage, optionally use `fetch_jira_issue` to pull additional context:
- Child issues (sub-tasks)
- Linked PRs (in comments or issue links)
- Recent comments added after the plan was generated

Place fetched context on the Blackboard. Proceed with plan execution regardless of whether
Jira fetch succeeds (graceful degradation).

## Communication

Use `comment_jira_issue` to post updates back to the Jira issue:
- After agent execution completes: post a summary of findings
- On close: post final status (success/failure/escalation)

Do NOT use `notify_gitlab_result` -- there is no GitLab MR for Jira events.
Do NOT use `refresh_gitlab_context` -- there is no MR state to refresh.

## Close Protocol

Jira events are autonomous -- no user confirmation needed.

1. Enter verify phase after agent work completes
2. Review agent findings against the plan's expected outcomes
3. Use `comment_jira_issue` to post results summary to the Jira issue
4. Close the event

No maintainer Slack notification needed -- the Jira issue IS the communication channel.
The human who approved the plan will see the results comment in their Jira notifications.

## What NOT To Do

- Do NOT call refresh_gitlab_context (no MR exists)
- Do NOT call notify_gitlab_result (no MR to comment on)
- Do NOT look for pipeline status (Jira issues don't have pipelines)
- Do NOT attempt to merge anything
- Do NOT dismiss GitLab todos
